# Test cases — diagram-derived user stories only

One table per sprint. Test cases cover the 43 diagram-defined user stories (US-1 … US-43) — **1 happy-path + 1 negative-path** per US, **86 total**. Off-diagram code (Exception A methods, the 8 consolidated boundaries, the debug page) is not in scope here — that's verified separately by the smoke tests under `tests/non_diagram/`.

The bold sentence at the start of each "Test Data" cell describes what the case verifies; what follows is the concrete input, written against the **seeded `app.db`** so a tester can read the TC and run it directly in the app. The "Actual Result" cell ends with a link to the **exact pytest test** backing that TC — clicking through opens the test source where the same `tc-…@a.com` / `fra_098` / etc. values appear. All 86 TCs are verified by the pytest suite (**412 tests** total pass on `revamp-final-diagrams` as of 2026-05-18; pytest covers every TC here plus additional edge cases).

**Reproducing these results:**

```bash
source .venv/bin/activate
pytest                          # 412 tests pass (full suite)
python scripts/verify_tcs.py    # TC-by-TC table → "Summary: 86/86 TCs pass"
```

### Backing test data — single shared scenario

Every TC's "Test Data" cell references rows seeded into `app.db` by `data/seed.py` — the 4 default-credential accounts at the front + the `TC - <>`-prefixed scenario rows at the trailing end of every table.

**Default credentials (acc_001..004, password `123`):**
- `a001@a.com` (acc_001, admin profile prof_001)
- `fr001@a.com` (acc_002, fundraiser profile prof_002)
- `d001@a.com` (acc_003, donee profile prof_003)
- `pm001@a.com` (acc_004, PM profile prof_004)

**TC scenario rows (acc_097..100, cat_098..100, fra_098..100, don_099..100, fav_100, rep_100, all with password `123`):**
- `tc-admin@a.com` — TC - Admin (acc_097)
- `tc-fr@a.com` — TC - Fundraiser (acc_098), owns `fra_098..100`
- `tc-d@a.com` — TC - Donee (acc_099), made `don_099..100` + the favourite `fav_100`
- `tc-pm@a.com` — TC - Platform Manager (acc_100), owns the monthly report `rep_100`
- Categories: `cat_098` "TC - Health", `cat_099` "TC - Education", `cat_100` "TC - Sports" (suspended)
- Activities: `fra_098` "TC - Active hospital fund" (active, ends 2027-12-31), `fra_099` "TC - Completed school fund" (`completed=1`, ends 2025-12-31), `fra_100` "TC - Suspended sports fund" (`suspended=1`)
- Donations: `don_099` ($100, `tc-d` → `fra_098`), `don_100` ($200, `tc-d` → `fra_099`)
- Favourite: `tc-d` → `fra_098`
- Report: `rep_100` monthly, April 2026, owned by `tc-pm`

Negative-path TCs (uniqueness collisions, FK violations, missing rows, cross-owner refusal) collide against these named rows or reference deliberately-unseeded IDs (`prof_999`, `acc_999`, `fra_999`, etc.).
