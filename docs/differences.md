# Diagrams ↔ Code: Catalogue of Differences

Compares every BCE class diagram and sequence diagram in [diagrams/](../diagrams/) against the current implementation. Generated by walking each `.jpg` per user story and reconciling against the code in [boundary/](../boundary/), [controller/](../controller/), and [entity/](../entity/).

Three classes of difference are tracked. Each entry tags one of:

- **[T] Diagram typo / internal inconsistency** — the diagram contradicts itself (class vs sequence) or has a spelling slip. Code follows the corrected version. Fix the diagram before final submission.
- **[D] Implementation deviation** — the code intentionally departs from what the diagram specifies (added parameters, changed semantics, different name). Diagram needs to be updated to match the code before final submission.
- **[E] Pragmatic Entity / Controller extension** — methods or fields exist in code but are not on any diagram. Sanctioned by [CLAUDE.md](../CLAUDE.md) Exception A. Diagrams to be extended before final submission.

For broader project context: [todo.md](todo.md) (deferred work + diagram catchup list), [issues.md](issues.md) (active design gaps), [implementation.md](implementation.md) (full code reference), [CLAUDE.md](../CLAUDE.md) (architecture rules).

## Sprint-wide / project-wide observations

These hold across every sprint:

- **Field type widening: `String` → typed.** Every diagram declares ID fields (`profileId`, `accountId`, `activityId`) as `String`. The code stores them as `int` (SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`). Per CLAUDE.md and the team's "no enum constraints on String fields" rule, `String` in diagrams is treated as a loose ID type — boundaries take text input but the persistent type is integer. **Not flagged below per use case.**
- **Method-name canonicalisation.** Diagrams use camelCase (e.g. `submitSearchCriteria`); Python uses snake_case (`submit_search_criteria`). Allowed by CLAUDE.md Rule 6. **Not flagged below.**
- **`Logout()` capitalisation.** Sequence diagrams for US-12, US-19, US-27, US-40 all draw the self-call as `Logout()` (capital L) while the class diagram has `logout()`. Code uses `logout()`. Logged once below under US-19; the same typo applies to all four logout sequences.
- **Trailing commas / cosmetic punctuation in diagram method signatures** (e.g. US-13 controller: `createFundraisingActivity(..., endDate: Date,): boolean`). Treated as cosmetic; not flagged per use case.
